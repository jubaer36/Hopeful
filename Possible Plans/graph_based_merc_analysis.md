# Graph-Based Approaches for Multimodal Emotion Recognition in Conversation (MERC)
## Comprehensive Analysis on IEMOCAP and MELD Datasets

> **Note on metric reporting**: Numbers vary across papers depending on (a) feature extractors used (e.g., sBERT vs RoBERTa for text, openSMILE vs Wav2Vec for audio, DenseNet/ResNet for video), (b) class settings (4-way vs 6-way IEMOCAP), and (c) whether full multimodal (A+V+T) or subset settings were used. Reported numbers below are the **multimodal (A+V+T) weighted F1 (WF1) and accuracy (Acc)** from each paper's primary table, with cross-referenced values where available.

| **GASMER / Adaptive Graph** | Topology learning sensitive to feature quality | Better pretrained encoders (RoBERTa, Wav2Vec2, HuBERT) | No class imbalance | **DER-GCN** minority contrastive | Limited MELD validation | Extensive MELD experiments with **GraphSmile** combo |

## Cross-Cutting Open Problems and Strongest Combinations

### 🚩 Major unsolved/under-solved problems in graph-based MERC

| Open Problem | Best Single-Paper Attempt | Strongest Multi-Paper Combination |
|---|---|---|
| **Modality dominance (text overshadows audio/video)** | AdaIGN / Ada²I | Ada²I + CausalMER + GraphCFC subspaces |
| **Class imbalance (rare: fear, disgust on MELD)** | DER-GCN (contrastive minority loss) | DER-GCN + VHGAE + CBERL |
| **Similar-emotion confusion (happy↔excited)** | AR-IIGCN, EACL | EACL anchors + CORECT P-CM |
| **Long-range conversation context** | Broad Mamba, GS-MCC | Broad Mamba + M3Net heterogeneous graphs |
| **Emotion shifts across turns** | GraphSmile, GAT-CRESA | GraphSmile + ConxGNN multi-scale |
| **Higher-order (>pairwise) cross-modal relations** | VHGAE, ConxGNN | VHGAE + HRG-SSA implicit edges |
| **Over-smoothing in deep GCNs** | MM-DFN, Joyful | Joyful contrastive + RBA-GCN filtering |
| **Multi-party scaling (MELD)** | GraphSmile, M3Net | GraphSmile + ConxGNN + Ada²I |
| **Early alignment noise** | MGLRA | MGLRA + CORECT P-CM |
| **Missing modality robustness** | (None strictly graph-based) | Integrate **CIDer** (causal+self-distillation) + **Multimodal Prompt Learning** with any graph backbone |

### 💡 Recommended Research Directions for Your Work

Based on this analysis, the most promising **untapped combinations** for novel research are:

1. **Hypergraph + Mamba + Emotion-Shift**: Combine VHGAE-style higher-order relations, Broad Mamba's efficient long-range modeling, and GraphSmile's emotion-shift dynamics. No existing paper merges all three.

2. **Causal Implicit Edge Inference**: HRG-SSA uses *similarity* for implicit edges; replace with **causal inference** (CausalMER-style counterfactuals) to discover edges based on emotional causation rather than surface similarity.

3. **LLM-Supervised Sentiment-Aware Graphs**: GraphSmile relies on noisy sentiment dynamics; use Emotion-LLaMA or NUS-Emo as a teacher to provide cleaner sentiment supervision, then feed into a graph backbone.

4. **Frequency-Decomposed Hypergraph**: GS-MCC uses binary low/high; extend to multi-band wavelet decomposition on hypergraph edges (combining GS-MCC + VHGAE).

5. **Class-Balanced Multi-Relational Graph with Modality Debiasing**: Combine DER-GCN's contrastive minority loss + M3Net's heterogeneous edges + Ada²I's gradient balancing + CausalMER debiasing — directly attacks the four hardest MELD problems (imbalance, multi-relation, modality dominance, spurious correlation).

6. **Generative Decoder with Hypergraph Encoder**: HRG-SSA shows generation works for MERC; combine with VHGAE encoder for higher-order context. Add anchor-based contrastive (EACL) on the generation side for similar-emotion separation.

## Notes on Reproducibility

- **Feature extractors matter**: numbers shift significantly based on text encoder (BERT vs RoBERTa vs sBERT) and audio encoder (openSMILE vs Wav2Vec2 vs HuBERT). Always check the reported feature pipeline before direct comparison.
- **IEMOCAP 4-way vs 6-way**: 4-way numbers (used by COGMEN, CORECT) are typically **15–20% higher** than 6-way because excited/frustrated are merged with happy/sad.
- **MELD train/val/test split**: standard split used across all papers above.
- **Multimodal (A+V+T) vs subset**: always check which modalities — some papers report best subset (e.g., A+T) as headline numbers.
