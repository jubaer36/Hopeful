# Graph-Based Approaches for Multimodal Emotion Recognition in Conversation (MERC)
## Comprehensive Analysis on IEMOCAP and MELD Datasets

> **Note on metric reporting**: Numbers vary across papers depending on (a) **feature extractors** (e.g., GloVe/TextCNN vs RoBERTa-Large vs sBERT vs T5 for text; OpenSmile IS10 vs Wav2Vec2 vs HuBERT for audio; DenseNet-FER+ vs OpenFace 2.0 vs 3D-CNN for video), (b) **class settings** (4-way vs 6-way IEMOCAP; standard vs MELD-1 protocol), (c) **fusion regime** (text-only vs multimodal early-concat vs late fusion vs graph fusion), and (d) **third-party re-implementations** which often differ from each paper's own table by 1–3 points. The numbers below are the **multimodal (A+V+T) weighted F1 (WF1) and accuracy (Acc)** as reported in each paper's primary table where available; we explicitly note when a figure originates from a third-party re-run.
>
> **Recurring author-group concentration**: A non-trivial fraction of 2023–2024 graph-based MERC methods come from three coordinated groups: (a) CSUFT-Hunan (Meng, Shou, Ai, Li) — DER-GCN, GS-MCC, Broad Mamba, MGLRA, TS-GCL, AR-IIGCN; (b) HUST (Jiang Li, Wang, Zeng) — GraphCFC, GraphMFT, GA²MIF, GraphSmile; (c) VNU-Vietnam (Nguyen, Le) — CORECT, Ada²I. This concentration means many "independent" methods share baselines and feature pipelines.

## Cross-Cutting Open Problems and Strongest Combinations

### 🚩 Major unsolved/under-solved problems in graph-based MERC

| Open Problem | Best Single-Paper Attempt | Strongest Multi-Paper Combination |
|---|---|---|
| **Modality dominance (text overshadows audio/video)** | **Ada²I** (AFW + AMW + disparity ratio) | Ada²I + causal debiasing + **GraphCFC** subspaces |
| **Class imbalance (rare: fear, disgust on MELD)** | **DER-GCN** (contrastive minority loss); **ConxGNN** (CB-focal contrastive) | DER-GCN + HAUCL + CB-focal contrastive |
| **Similar-emotion confusion (happy↔excited)** | **AR-IIGCN**; EACL anchor-based | EACL anchors + **CORECT** P-CM |
| **Long-range conversation context** | **Broad Mamba** (SSM); **GS-MCC** (Fourier) | Broad Mamba + **M3Net** hypergraph |
| **Emotion shifts across turns** | **GraphSmile** (SDP auxiliary); **GAT-CRESA** | GraphSmile + **ConxGNN** multi-scale |
| **Higher-order (>pairwise) cross-modal relations** | **M3Net** (hypergraph); **HAUCL** (variational hypergraph) | HAUCL + **HRG-SSA** implicit edges |
| **Over-smoothing in deep GCNs** | **MMGCN** (GCNII tricks); **GraphCFC** (GAT-MLP); **Joyful** (contrastive) | Joyful contrastive + **RBA-GCN** filtering |
| **Multi-party scaling (MELD)** | **GraphSmile**, **M3Net** | GraphSmile + ConxGNN + Ada²I |
| **Early alignment noise** | **MGLRA** (masked GCN + recurrent alignment) | MGLRA + **CORECT** P-CM |
| **Missing modality robustness** | (None strictly graph-based) | Integrate **CIDer** (causal+self-distillation) or Multimodal Prompt Learning with any graph backbone |
| **Online / real-time MERC** | (Open — all current graph methods are offline) | Buffered context window + dynamic graph construction |

### 💡 Recommended Research Directions

Based on this analysis, the most promising **untapped combinations** for novel research are:

1. **Hypergraph + Mamba + Emotion-Shift**: Combine HAUCL/VHGAE-style higher-order relations, Broad Mamba's efficient long-range modeling, and GraphSmile's SDP emotion-shift dynamics. No existing paper merges all three.

2. **Causal Implicit Edge Inference**: HRG-SSA uses *similarity* for implicit edges; replace with **causal inference** (counterfactual reasoning) to discover edges based on emotional causation rather than surface similarity.

3. **LLM-Supervised Sentiment-Aware Graphs**: GraphSmile's SDP relies on noisy utterance-level sentiment; use Emotion-LLaMA or similar as a teacher to provide cleaner sentiment supervision, then feed into a graph backbone.

4. **Frequency-Decomposed Hypergraph**: GS-MCC uses binary low/high frequency; extend to multi-band wavelet decomposition on hypergraph edges (combining GS-MCC + HAUCL).

5. **Class-Balanced Multi-Relational Graph with Modality Debiasing**: Combine DER-GCN's contrastive minority loss + M3Net's hypergraph + Ada²I's modality weighting + causal debiasing — directly attacks the four hardest MELD problems (imbalance, multi-relation, modality dominance, spurious correlation).

6. **Generative Decoder with Hypergraph Encoder**: HRG-SSA shows T5-generation works for MERC; combine with HAUCL hypergraph encoder for higher-order context. Add anchor-based contrastive (EACL) on the generation side for similar-emotion separation.

7. **Online Graph Construction with Bounded Buffer**: All current methods are offline. Combine **streaming graph updates** with **GS-MCC**-style Fourier operators or **Broad Mamba** SSM for real-time MERC suitable for live applications.

## Notes on Reproducibility and Comparison Validity

- **Feature extractors matter significantly**: Numbers shift by 3–7 points based on text encoder alone (GloVe vs RoBERTa vs sBERT vs T5). Always check the reported feature pipeline before direct comparison. The MMGCN family (DialogueGCN/MMGCN/MM-DFN) uses TextCNN+GloVe; M3Net uses RoBERTa-Large fine-tuned; HRG-SSA uses T5-base; COGMEN uses sBERT.
- **IEMOCAP 4-way vs 6-way**: 4-way numbers (COGMEN, CORECT primary table) are typically 15–20% higher than 6-way because excited/frustrated are merged with happy/sad.
- **MELD-1 vs standard MELD protocol**: Some papers (e.g., GraphSmile) report numbers under a non-standard MELD-1 protocol that yields substantially higher F1 (~74 vs ~67). Always verify protocol.
- **Multimodal (A+V+T) vs subset**: Some papers report best subset (e.g., A+T) as headline numbers. The bimodal A+T often outperforms full A+V+T on IEMOCAP because visual modality is noisy.
- **Original vs re-implementation**: Numbers for older methods (DialogueGCN, MMGCN) in newer papers' baseline tables are typically **re-implementations** that differ from the original paper by 1–3 points. The HRG-SSA paper (IJCAI 2025) is a good reference for consistent third-party re-implementation under a unified pipeline.
- **Loss function differences**: MMGCN and MM-DFN use focal loss on MELD for class imbalance; DialogueGCN uses plain cross-entropy. ConxGNN uses CB-focal contrastive. This affects MELD comparisons.
- **Train/val/test splits**: Standard IEMOCAP uses sessions 1–4 for training, session 5 for test (often with 10% of training as validation). MELD uses the official train/val/test split. Some papers deviate — always verify.
- **Statistical significance**: Many reported gains of <1 WF1 may not be statistically significant. Only some papers (MM-DFN, MMGCN) report paired t-tests.
